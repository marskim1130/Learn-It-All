聊天会话 Demo

演示创建、列表、详情与数据隔离： POST /conversations、GET /conversations、 GET /conversations/:id。

API

POST /conversations
{ "title": "学习计划" }  // title 可选，默认“新会话”
→ 201 { conversation }

GET /conversations
→ 200 { conversations: [...] }  // 仅当前用户

GET /conversations/:id
→ 200 自己的会话 / 404 NOT_FOUND（他人或不存在）

PATCH /conversations/:id
{ "title": "新标题" }
→ 200 更新后的会话；空标题 400；他人 404

DELETE /conversations/:id
→ 204；列表与详情不再可见；他人 404

设计思想

  • 所有查询绑定 owner_id，授权在服务端完成。
  • 跨用户与不存在统一 404，避免资源枚举。
  • 仓储接缝便于内存测试与 PostgreSQL 实现切换。
  • 未来 messages 表应对 conversation_id 使用 ON DELETE CASCADE。

常见错误

  • 未登录就创建，应得到 401。
  • 用别人的会话 ID 访问详情，应得到 404 而不是 403。
  • 前端按 owner 过滤列表，却不在 API 侧约束。
  • 空标题重命名应得到 400 VALIDATION_ERROR。
