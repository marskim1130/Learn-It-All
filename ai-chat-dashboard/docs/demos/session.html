登录与身份会话 Demo

演示 POST /auth/login、GET /auth/me、 POST /auth/logout：HttpOnly Cookie、统一错误凭据、退出后会话失效。

API

POST /auth/login
{ "email": "alice@example.com", "password": "password123" }
→ 200 + Set-Cookie: access_token=...; HttpOnly; SameSite=Lax

GET /auth/me
Cookie: access_token=...
→ 200 { user } 或 401 UNAUTHORIZED

POST /auth/logout
→ 204 + 清空 Cookie；旧令牌立即失效

设计思想

  • 错误凭据统一 INVALID_CREDENTIALS，避免账户枚举。
  • 令牌只走 Cookie，不进 JSON 响应体。
  • 服务端会话表保证退出后旧 Cookie 不能继续使用。

可运行 Demo

先启动 API（默认 http://localhost:3001）。浏览器需允许跨站 Cookie 时注意同源或 CORS 凭证；本页默认请求同源 API 地址。

常见错误

  • 前端把 token 存 localStorage，扩大 XSS 风险。
  • 错误密码与邮箱不存在返回不同文案。
  • 只清 Cookie、服务端不删会话，旧令牌仍有效。
